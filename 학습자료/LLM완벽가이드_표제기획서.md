**[이 원고의 차별화 포지션 확정]**
"LLM을 API로 쓰는 법"이 아니라 **"LLM을 직접 학습시키고, 추론하게 만들고, 서비스로 내보내는 법"**을 다루는 2026년형 모델 엔지니어링 종합서. 텍스트 SFT → VLM → 펑션콜링 튜닝 → 서빙 → GRPO 추론 학습 → RAG·에이전트로 이어지는 단권 파이프라인은 국내 공백이며, 표제는 이 "학습시키는 사람의 책"이라는 정체성과 Reasoning 키워드를 전면에 세워야 한다.

### ⚠️ 비판적·보수적 시장성 검토 (별도 편성)

- **시장 포화 리스크: 중.** 'LLM'이 제목에 들어간 책은 다수이나, 본서의 실질 경쟁 구간(학습~서빙)은 2권뿐. 다만 허정준 책이 개정판을 내면 정면충돌하므로 선점 속도가 중요.
- **독자 규모 한계: 중.** "직접 파인튜닝하는 개발자"는 "API 쓰는 개발자"보다 확실히 좁다. 다만 경쟁서 2권이 이 좁은 시장에서 성과를 냈다는 사실 자체가 시장 규모의 하한 증명. 부제에 RAG·에이전트를 명시해 응용 수요를 함께 흡수하는 설계 필요.
- **기술 유효기간: 높음(주의).** Qwen3, LangChain 1.x, transformers 4.5x는 12~18개월 내 구세대화 가능. 저장소 기반 버전 대응 체계(requirements 고정 + env-updates 문서)를 마케팅 포인트로 전환할 것.
- **번역서 경쟁: 중.** 핸즈온 LLM·밑바닥 LLM이 원리 영역을 점유했으나, 두 책 모두 GRPO·VLM 튜닝·한국어 실습이 없다. 번역 시차가 본서의 방어벽.
- **출판 타이밍: 양호~긴박.** DeepSeek-R1발 Reasoning 관심은 2025년 정점 후에도 실무 채택기로 이행 중. 다만 같은 공백을 노리는 국내 기획이 진행 중일 확률이 높아, **2026년 내 출간이 포지션 유지의 조건.**

### ★ 추천 조합안 (3세트)

**조합 1 (최우선 추천): 『생각하는 LLM 만들기』 — LoRA 파인튜닝부터 VLM, DeepSeek-R1 방식 추론 모델 학습, vLLM 서빙, RAG 에이전트까지**
- 왜 팔리는가: '생각하는'은 Reasoning LLM이라는 이 책의 유일무이한 차별점을 비전공자 눈에도 읽히는 일상어로 번역한다. o1·DeepSeek-R1 뉴스로 "AI가 생각한다"는 표현에 이미 노출된 독자층에게 즉시 연결되고, "만들기"는 이 책이 활용서가 아니라 구축서임을 선언한다. 부제가 파인튜닝·vLLM·RAG 등 검색 키워드를 전부 수거해 온라인 서점 노출을 보완한다.
- 주 타깃: LLM 활용 단계를 졸업하고 모델 내부로 들어가려는 1~5년차 개발자
- 직접 경쟁 대비 우위 1줄: 『실전 LLM 파인튜닝』이 멈춘 지점(SFT)에서 시작해 추론 모델 학습까지 가는 유일한 책.

**조합 2: 『LLM 풀스택』 — 파인튜닝, 서빙 최적화, RAG 에이전트, 그리고 추론 모델 직접 만들기**
- 왜 팔리는가: '풀스택'은 개발자 시장에서 검증된 자기 정체성 단어로, "한 권으로 끝내는" 류의 포화 표현을 쓰지 않고도 완결성을 전달한다. 짧고(6자) 기억되며, "LLM 풀스택 개발자"라는 커리어 열망 프레임을 만든다. 커뮤니티(FB 그룹, 커리어리)에서 해시태그화되기 쉬운 제목.
- 주 타깃: 커리어 확장을 노리는 백엔드/ML 엔지니어, 이직 준비 개발자
- 직접 경쟁 대비 우위 1줄: 허정준 책의 커버리지에 Reasoning·VLM·펑션콜링 튜닝을 더한 2026년판 상위 호환 포지션.

**조합 3: 『나만의 LLM 만들기』 — 한국어 데이터로 완주하는 파인튜닝·추론·서빙 파이프라인**
- 왜 팔리는가: sLLM 사내 구축·도메인 특화 튜닝이라는 2026년 기업 수요를 '나만의'라는 소유격으로 직역한다. 입문 친화적 어감이라 진입 장벽 인상을 낮추면서, 부제의 '한국어 데이터 완주'가 번역서가 절대 못 주는 가치를 명시한다.
- 주 타깃: 사내 특화 모델 구축 미션을 받은 실무자, 도메인 데이터 보유 기업의 개발자
- 직접 경쟁 대비 우위 1줄: 핸즈온 LLM 등 번역서가 채우지 못하는 '우리 데이터로 우리 모델' 수요의 국내 유일 응답.

### 온라인 서점 소개 카피

**200자 버전 (예스24·교보 짧은 소개용)**
> ChatGPT를 '쓰는' 법은 이제 누구나 안다. 이 책은 LLM을 '만드는' 법을 다룬다. LoRA 파인튜닝, Qwen3-VL 멀티모달 튜닝, DeepSeek-R1 방식의 추론 모델 학습, vLLM 서빙, RAG 에이전트까지 — 한국어 데이터로 처음부터 끝까지 직접 완주하는 2026년형 LLM 엔지니어링 실전서.

**500자 버전 (상세 소개·상품 페이지용)**
> "파인튜닝 책 따로, RAG 책 따로, 서빙은 공식 문서로." — LLM을 제대로 다루려는 개발자가 겪는 현실이다. 이 책은 그 조각들을 하나의 파이프라인으로 잇는다.
>
> 언어 모델의 밑바닥 원리와 트랜스포머 구현에서 출발해, LoRA/QLoRA로 한국어 뉴스 모델을 파인튜닝하고, Qwen3-VL로 이미지를 이해하는 모델을 만들고, 쇼핑몰 챗봇에 펑션콜링을 학습시킨다. 이어 Flash Attention과 연속 배치로 서빙 병목을 해부하고, 마침내 DeepSeek-R1의 GRPO 방법론으로 '생각하는' 추론 모델을 직접 학습시킨다. 마지막은 금융 PDF RAG 챗봇과 Text-to-SQL 에이전트 — 학습시킨 모델을 실무 서비스로 내보내는 단계다.
>
> 모든 실습은 한국어 데이터와 2026년 최신 스택(Qwen3, transformers 4.5x, LangChain 1.x)으로 검증했다. 모델을 빌려 쓰는 개발자에서, 모델을 만들어내는 엔지니어로 — 그 전환을 위한 가장 완결된 한 권이다.

### 표지·띠지 후킹 문장 (3개)
1. "o1과 DeepSeek-R1은 어떻게 '생각'하게 됐을까? 이제 당신의 GPU에서 직접 재현한다."
2. "API를 호출하는 개발자와, 모델을 학습시키는 개발자. 연봉 테이블이 갈리는 지점."
3. "파인튜닝에서 멈춘 책은 많다. 추론 모델 학습과 서빙까지 데려가는 책은 이 책뿐이다."

### 영업·홍보팀 셀링포인트 (3가지)
1. **국내 첫 추론 모델(Reasoning LLM) 학습서** — DeepSeek-R1의 GRPO 방법론을 이론이 아니라 MiniR1 구현 프로젝트로 완주한다. 경쟁 도서 《한 권으로 끝내는 실전 LLM 파인튜닝》과 비교 시 우위: SFT에서 끝나는 경쟁서와 달리 강화학습 기반 추론 학습까지 도달하는 유일한 국내서.
2. **텍스트를 넘어 멀티모달·에이전트 튜닝까지 단권 커버** — Qwen3-VL 파인튜닝과 펑션콜링 '학습'(사용이 아님)을 모두 수록. 경쟁 도서 《LLM을 활용한 실전 AI 애플리케이션 개발》과 비교 시 우위: 2024년 스택 기반 선행서가 다루지 못한 VLM·Reasoning·펑션콜링 튜닝을 2026년 스택으로 수록한 상위 호환.
3. **한국어 데이터 완주 + 재현성 설계** — 뉴스·쇼핑몰·금융 PDF 등 전 실습이 국내 데이터이며, RunPod 이미지·라이브러리 버전까지 고정 명시. 경쟁 도서 《핸즈온 LLM》(번역서)과 비교 시 우위: 번역서가 구조적으로 제공 불가능한 한국어 도메인 실습과 국내 GPU 환경 가이드를 제공.
